# 投递说明

当报告需要由定时任务推送到飞书时，遵循这份说明。

## 投递步骤

1. 使用 `scripts/render_feishu_report.py` 渲染最终报告。
2. 把渲染结果视为唯一可信的消息源：
   - `title`：消息标题
   - `markdown`：消息正文
   - `metadata`：用于日志、去重或追踪的机器可读上下文
3. 检查当前运行环境是否暴露了定时任务入口，或其下游飞书发送 connector。
4. 按任务或 connector 的真实参数名映射渲染结果中的字段。
5. 只有在定时执行链路明确返回成功后，才视为发送完成。

## 运行时规则

- 优先使用已有的定时任务入口、MCP server 或内置 connector，不要手写未经确认的 HTTP 调用。
- 不要编造调度 endpoint、任务参数、请求体或飞书消息格式。
- 如果任务或 connector 需要接收人标识，使用用户提供的飞书目标，或环境里已有的默认配置。
- 在标题中带上报告日期，减少重复发送的风险。

## 失败处理

- 如果缺少调度配置或投递凭证，就在渲染完成后停止，并返回 payload。
- 如果定时任务或下游 connector 调用失败，要明确说明阻塞点，并保留原始渲染结果不变。
- 除非工具返回明确成功，否则不要宣称消息已经推送到飞书。
